Jellyfin — Lokaler Medienserver

Medien 30–45 Minuten
Windows Linux macOS Docker Raspberry Pi
Fortschritt0%

Was ist das?

Für Kinder erklärt

Jellyfin ist wie dein eigenes Netflix zu Hause! Du legst deine Filme, Serien und Musik auf den Computer, und dann kannst du sie auf dem Fernseher, Tablet oder Handy anschauen — ganz ohne Internet!

Einfach erklärt

Jellyfin ist ein kostenloser, quelloffener Medienserver. Du installierst ihn auf einem Computer und er organisiert deine Filme, Serien, Musik und Fotos in einer schönen Oberfläche — ähnlich wie Netflix oder Spotify. Alle Geräte in deinem Heimnetzwerk können darauf zugreifen. Anders als Plex braucht Jellyfin kein Konto und keine Cloud-Verbindung. Alles bleibt lokal bei dir.

Technische Details

Jellyfin ist ein Fork von Emby und unter der GPL-v2-Lizenz veröffentlicht. Der Server ist in C# (.NET) geschrieben und bietet eine REST-API, Hardware-Transkodierung (VAAPI, NVENC, QSV), DLNA-Unterstützung, Live-TV mit DVB-Tunern, Benutzerverwaltung mit Zugriffsrechten und Plugin-System. Unterstützte Medienformate umfassen nahezu alle gängigen Video- (H.264, H.265, VP9, AV1), Audio- (FLAC, MP3, AAC, Opus) und Bildformate.

Installation

1

Jellyfin herunterladen

Öffne deinen Browser und gehe zu jellyfin.org. Klicke auf "Download" und wähle den Windows-Installer. Die Datei ist ca. 150 MB groß.

2

Installer ausführen

Doppelklicke auf die heruntergeladene jellyfin_xxx_windows-x64.exe. Folge dem Installationsassistenten und wähle "Als Dienst installieren", damit Jellyfin automatisch beim Hochfahren startet.

Häufige Fehler

Windows Defender warnt: Klicke auf "Weitere Informationen" und dann "Trotzdem ausführen". Jellyfin ist sicher und Open Source.

Port belegt: Falls Port 8096 bereits verwendet wird, kannst du den Port in der Konfigurationsdatei ändern.

3

Weboberfläche öffnen

Öffne deinen Browser und gehe zu:

Browser 
http://localhost:8096

Der Einrichtungsassistent startet automatisch beim ersten Zugriff.

4

Ersteinrichtung durchführen

Wähle deine Sprache (Deutsch), erstelle einen Admin-Benutzer mit Passwort und klicke auf "Weiter". Die Medienbibliotheken richtest du im nächsten Abschnitt ein.

1

Repository hinzufügen und installieren

Debian / Ubuntu:

Terminal
curl https://repo.jellyfin.org/install-debuntu.sh | sudo bash

Fedora / RHEL:

Terminal
# Jellyfin-Repository hinzufügen
sudo dnf install https://repo.jellyfin.org/releases/server/fedora/stable/jellyfin-server-latest.noarch.rpm

# Jellyfin installieren
sudo dnf install jellyfin jellyfin-web
2

Dienst starten

Terminal
sudo systemctl enable jellyfin
sudo systemctl start jellyfin
3

Weboberfläche öffnen

Browser
http://localhost:8096
Häufige Fehler

Zugriff verweigert auf Medienordner: Jellyfin läuft als Benutzer jellyfin. Stelle sicher, dass dieser Benutzer Leserechte auf deine Medienordner hat: sudo chmod -R o+rx /pfad/zu/medien

Firewall blockiert: Öffne den Port mit sudo ufw allow 8096/tcp

1

Jellyfin installieren

Nutze Homebrew oder lade die DMG von jellyfin.org herunter:

Terminal
brew install --cask jellyfin
Gatekeeper-Warnung

Beim ersten Start fragt macOS, ob du die App wirklich öffnen willst. Gehe zu Systemeinstellungen → Datenschutz & Sicherheit und klicke auf "Trotzdem öffnen".

2

Jellyfin starten

Starte Jellyfin über die App oder das Terminal:

Terminal
jellyfin

Öffne dann http://localhost:8096 im Browser.

1

Docker-Image herunterladen

Terminal
docker pull jellyfin/jellyfin
2

Container starten

Terminal
docker run -d \
  --name jellyfin \
  --restart=unless-stopped \
  -p 8096:8096 \
  -v /pfad/zu/config:/config \
  -v /pfad/zu/cache:/cache \
  -v /pfad/zu/medien:/media \
  jellyfin/jellyfin

Ersetze /pfad/zu/medien mit dem tatsächlichen Pfad zu deinen Medien. Öffne dann http://localhost:8096 im Browser.

3

Docker Compose (Alternative)

docker-compose.yml
services:
  jellyfin:
    image: jellyfin/jellyfin
    container_name: jellyfin
    restart: unless-stopped
    ports:
      - "8096:8096"
    volumes:
      - ./config:/config
      - ./cache:/cache
      - /pfad/zu/medien:/media
Terminal
docker compose up -d
1

Jellyfin-Repository hinzufügen

Auf Raspberry Pi OS (Debian-basiert) nutze das offizielle Installationsskript:

Terminal
curl https://repo.jellyfin.org/install-debuntu.sh | sudo bash
2

Dienst starten

Terminal
sudo systemctl enable jellyfin
sudo systemctl start jellyfin

Öffne auf einem anderen Gerät im Netzwerk http://<raspberry-pi-ip>:8096.

3

IP-Adresse herausfinden

Terminal
hostname -I

Die erste angezeigte IP-Adresse ist die deines Raspberry Pi. Nutze sie mit Port 8096 im Browser.

Tipp für den Raspberry Pi

Ein Raspberry Pi 4 mit 4 GB RAM eignet sich gut als Jellyfin-Server. Hardware-Transkodierung wird allerdings nur eingeschränkt unterstützt. Am besten speicherst du Medien in Formaten, die deine Endgeräte direkt abspielen können (Direct Play).

Bibliothek in Jellyfin hinzufügen

1

Gehe im Einrichtungsassistenten (oder unter Dashboard → Bibliotheken) auf "Medienbibliothek hinzufügen".

2

Wähle den Inhaltstyp (Filme, Serien, Musik oder Fotos) und füge den Ordnerpfad hinzu.

3

Aktiviere die gewünschten Metadaten-Quellen (z.B. TheMovieDB für Filme und Serien). Jellyfin lädt automatisch Cover, Beschreibungen und Bewertungen herunter.

4

Klicke auf "OK" und warte, bis der Bibliotheksscan abgeschlossen ist. Je nach Medienmenge kann das einige Minuten dauern.

Tipp: Benenne Filme und Serien nach dem Schema "Name (Jahr)" — so findet Jellyfin automatisch die richtigen Metadaten. Für Serien nutze das Format S01E01 in den Dateinamen.

Transkodierung einrichten

Transkodierung wandelt Videos in Echtzeit um, wenn ein Gerät das Originalformat nicht abspielen kann. Hardware-Transkodierung ist deutlich schneller als Software-Transkodierung.

1

Gehe zu Dashboard → Wiedergabe → Transkodierung.

2

Wähle deine Hardware-Beschleunigung:

HardwareEinstellungHinweis
NVIDIA GPUNVENCBenötigt aktuelle NVIDIA-Treiber
Intel iGPUIntel Quick Sync (QSV)Auf den meisten Intel-CPUs verfügbar
AMD GPUVA-APILinux bevorzugt, unter Windows eingeschränkt
Keine GPUKeineSoftware-Transkodierung (langsamer, mehr CPU-Last)
3

Aktiviere die gewünschten Codecs (H.264 und H.265 sind am häufigsten) und klicke auf "Speichern".

Direct Play vs. Transkodierung

Direct Play: Das Endgerät spielt die Datei direkt ab, ohne Umwandlung. Das ist am effizientesten und belastet den Server kaum.

Transkodierung: Der Server wandelt das Video in Echtzeit um. Das ist nötig, wenn das Endgerät das Format nicht unterstützt (z.B. HEVC auf älteren Geräten).

Empfehlung: Speichere Medien wenn möglich in H.264 (MP4/MKV) — dieses Format wird von fast allen Geräten direkt unterstützt.

Deine Schritte

  • Jellyfin installiert
  • Ersteinrichtung abgeschlossen (Admin-Benutzer erstellt)
  • Medienbibliothek hinzugefügt (Filme, Serien oder Musik)
  • Im Browser auf Medien zugegriffen
  • Client-App auf einem Gerät eingerichtet
  • Transkodierung konfiguriert (optional)

Geschafft!

Du hast jetzt deinen eigenen Medienserver — kostenlos, ohne Cloud, ohne Konto. Deine Filme, Serien und Musik stehen dir auf allen Geräten in deinem Netzwerk zur Verfügung.
