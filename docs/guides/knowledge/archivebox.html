ArchiveBox — Webseiten dauerhaft offline archivieren

Wissen Sichern Mittel 20–30 Minuten
Windows (WSL) Linux macOS Docker
Fortschritt0%

Was ist das?

Fur Kinder erklärt

ArchiveBox ist wie ein magisches Fotoalbum für Webseiten — es macht einen Schnappschuss von jeder Seite und bewahrt ihn auf, damit du ihn auch ohne Internet jederzeit anschauen kannst!

Einfach erklärt

ArchiveBox ist ein kostenloses, selbst-gehostetes Werkzeug, mit dem du komplette Webseiten auf deinem eigenen Computer speicherst. Es sichert nicht nur den Text, sondern auch Bilder, CSS, JavaScript, PDFs und sogar Screenshots. Wenn eine Webseite später offline geht oder verändert wird, hast du immer noch deine eigene Kopie. Stell es dir vor wie eine persönliche Wayback Machine.

Technische Details

ArchiveBox ist eine Open-Source-Software (archivebox.io), die Webseiten in mehreren Formaten gleichzeitig archiviert: HTML mit allen Assets, Screenshot (PNG), PDF, WARC (Web ARChive — das Standardformat für Web-Archive), eingebettete Medien und sogar Git-Repositories. Es nutzt unter anderem wget, Chrome/Chromium (headless) und youtube-dl im Hintergrund. Die Daten werden in einem durchsuchbaren Index mit Web-Oberfläche gespeichert. ArchiveBox kann per CLI, Web-UI oder REST-API gesteuert werden.

Installation

1

Docker installieren

Falls Docker noch nicht installiert ist, lade es von docker.com herunter. Unter Linux kannst du es so installieren:

Terminal
sudo apt install docker.io docker-compose
2

Projektverzeichnis anlegen

Terminal
mkdir ~/archivebox && cd ~/archivebox
3

Docker Compose Datei erstellen

Erstelle eine docker-compose.yml Datei mit folgendem Inhalt:

docker-compose.yml
services:
  archivebox:
    image: archivebox/archivebox:latest
    ports:
      - 8000:8000
    volumes:
      - ./data:/data
    environment:
      - ALLOWED_HOSTS=*
      - MEDIA_MAX_SIZE=750m
4

ArchiveBox initialisieren und starten

Terminal
# Datenbank initialisieren
docker compose run archivebox init

# Admin-Benutzer erstellen
docker compose run archivebox manage createsuperuser

# Server starten
docker compose up -d

Öffne dann im Browser: http://localhost:8000 — dein persönliches Web-Archiv ist bereit!

Webseiten archivieren

1

Einzelne URL archivieren

Der einfachste Weg, eine Webseite zu sichern:

Terminal
archivebox add https://example.com/wichtiger-artikel

In Docker:

Terminal
docker compose run archivebox add https://example.com/wichtiger-artikel
2

Lesezeichen importieren

Du kannst eine ganze Liste von URLs oder exportierte Browser-Lesezeichen importieren:

Terminal
# Aus einer Textdatei mit URLs (eine pro Zeile)
archivebox add < urls.txt

# Browser-Lesezeichen importieren (HTML-Export)
archivebox add < bookmarks.html

Tipp: Exportiere deine Lesezeichen in Chrome unter Lesezeichen-Manager > Drei Punkte > Lesezeichen exportieren. In Firefox unter Lesezeichen > Lesezeichen verwalten > Importieren und Sichern > Lesezeichen nach HTML exportieren.

3

Automatische Archivierung planen

Richte einen Cronjob ein, der regelmäßig neue Inhalte archiviert:

Terminal
# Jeden Tag um 3 Uhr nachts eine RSS-Feed-URL archivieren
crontab -e

# Diese Zeile hinzufügen:
0 3 * * * cd ~/archivebox && archivebox add https://example.com/rss.xml --depth=1

Speicherplatz beachten: Jede archivierte Seite kann 5–50 MB beanspruchen (mit Screenshots und PDFs). Plane genug Festplattenspeicher ein, wenn du viele Seiten archivierst!

Was wird gespeichert?

ArchiveBox speichert jede Webseite automatisch in mehreren Formaten gleichzeitig:

FormatBeschreibungVorteil
HTMLKomplette Seite mit CSS, JS und BildernOriginalgetreue Darstellung im Browser
ScreenshotBildschirmfoto der gesamten Seite (PNG)Visueller Beweis, unabhängig vom Browser
PDFDruckversion als PDF-DateiEinfach zu teilen und auszudrucken
WARCWeb ARChive — Standard-ArchivformatVollständigste Kopie, institutioneller Standard
MediaVideos, Audio und andere MedienEingebettete Inhalte werden mitgesichert
GitGit-Repositories klonenQuellcode komplett mit Versionshistorie

Konfiguration

ArchiveBox lässt sich über Umgebungsvariablen oder die Datei ArchiveBox.conf konfigurieren. Hier die wichtigsten Einstellungen:

ArchiveBox.conf
# Welche Formate gespeichert werden sollen
SAVE_WGET=True
SAVE_PDF=True
SAVE_SCREENSHOT=True
SAVE_WARC=True
SAVE_GIT=True
SAVE_MEDIA=True

# Maximale Medien-Dateigröße
MEDIA_MAX_SIZE=750m

# Zeitlimit pro Archivierung (in Sekunden)
TIMEOUT=60

# Nur bestimmte Formate deaktivieren, um Speicher zu sparen
SAVE_MEDIA=False
SAVE_GIT=False

Tipp: Wenn du Speicherplatz sparen willst, deaktiviere SAVE_MEDIA und SAVE_GIT. Die wichtigsten Formate (HTML, Screenshot, PDF) reichen für die meisten Anwendungsfälle völlig aus.

  • Admin-Benutzer erstellt
  • Erste Webseite archiviert
  • Web-Oberfläche im Browser geöffnet

  • Automatische Archivierung eingerichtet (optional)
