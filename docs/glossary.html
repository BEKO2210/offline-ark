Glossar

Alle wichtigen Begriffe verständlich erklärt — von A bis Z.

A B C D E F G H J K L M N O P Q R S T U V W Y Z

A

AES-256

Ein extrem sicherer Verschlüsselungsstandard. Wird von KeePassXC und VeraCrypt genutzt, um deine Daten zu schützen. Selbst die schnellsten Computer der Welt könnten die Verschlüsselung nicht knacken.

Technisch
Advanced Encryption Standard mit 256-Bit Schlüssellänge. Symmetrische Blockchiffre, vom NIST standardisiert. 2^256 mögliche Schlüssel.

API (Application Programming Interface)

Eine Schnittstelle, über die Programme miteinander reden können. Zum Beispiel kann ein Chat-Programm über die Ollama-API mit der KI sprechen.

Technisch
Definierte Endpunkte für Programm-zu-Programm-Kommunikation. Ollama bietet eine REST-API auf Port 11434 mit JSON-Anfragen/Antworten.

AppImage

Ein Dateiformat für Linux-Programme, die du einfach herunterladen und starten kannst — ohne Installation. Wie eine .exe für Linux.

Technisch
Portable Linux-Anwendung, die alle Abhängigkeiten in einer Datei bündelt. Benötigt nur chmod +x zum Ausführen. Distributionsunabhängig.

B

Backup

Eine Sicherungskopie deiner Daten. Wenn dein Computer kaputt geht, hast du noch eine Kopie. Die 3-2-1-Regel: 3 Kopien, 2 verschiedene Medien, 1 außer Haus.

Bluetooth

Eine Funktechnik für kurze Distanzen (bis ca. 10 Meter). Wird von Briar Messenger genutzt, um ohne Internet zu kommunizieren.

C

CLI (Command Line Interface)

Die Kommandozeile — ein Textfenster, in das du Befehle eingibst. Sieht altmodisch aus, ist aber oft der schnellste Weg, etwas zu erledigen.

Technisch
Textbasierte Benutzerschnittstelle. Windows: PowerShell/CMD. Linux/macOS: Terminal/Bash/Zsh. Viele Offline-Tools werden über CLI gesteuert.

Cloud

Computer anderer Leute, auf denen deine Daten gespeichert werden. Google Drive, iCloud, Dropbox — das ist alles "Cloud". Offline Ark zeigt dir Alternativen, bei denen deine Daten bei dir bleiben.

Container / Docker

Eine Art "virtueller Karton", in dem ein Programm mit allen Dingen läuft, die es braucht. Macht Installation einfacher, weil alles fertig gepackt ist.

Technisch
OS-Level-Virtualisierung. Docker-Container teilen sich den Host-Kernel, sind aber voneinander isoliert. Images werden von Docker Hub oder eigenen Registries gezogen.

D

Datenbank

Ein strukturierter Speicher für Daten. Deine KeePassXC-Datei ist eine verschlüsselte Datenbank. Webseiten nutzen Datenbanken, um Inhalte zu speichern.

DNS (Domain Name System)

Das "Telefonbuch des Internets". Wandelt Adressen wie "wikipedia.org" in Nummern um, die Computer verstehen. Offline brauchst du kein DNS.

E

Encryption / Verschlüsselung

Daten so umwandeln, dass nur du sie lesen kannst. Wie ein Geheimcode — ohne den Schlüssel sieht alles aus wie Kauderwelsch.

F

Flatpak

Ein System, um Programme unter Linux zu installieren. Funktioniert auf jeder Linux-Version gleich. Ähnlich wie ein App Store.

G

GGUF

Ein Dateiformat für KI-Modelle, das von Ollama und anderen Tools verwendet wird. Steht für "GPT-Generated Unified Format".

Technisch
Nachfolger von GGML. Optimiert für Inference auf Consumer-Hardware. Unterstützt verschiedene Quantisierungsstufen (Q4, Q5, Q8, F16).

GPU (Grafikkarte)

Der Chip in deinem Computer, der normalerweise Bilder und Spiele berechnet. KI-Modelle laufen auf GPUs viel schneller als auf dem normalen Prozessor.

Technisch
Graphics Processing Unit. NVIDIA GPUs nutzen CUDA, AMD nutzt ROCm, Apple Silicon nutzt Metal. VRAM (Video-RAM) bestimmt, wie große Modelle geladen werden können.

H

Homelab

Ein kleines "Rechenzentrum" bei dir zu Hause. Kann so einfach sein wie ein Raspberry Pi, der als Server läuft.

J

JSON

Ein Textformat, in dem Programme Daten austauschen. Sieht aus wie eine strukturierte Liste mit geschweiften Klammern.

K

KI / AI (Künstliche Intelligenz)

Programme, die "intelligent" reagieren können — Texte schreiben, Fragen beantworten, Bilder erstellen. Mit Ollama kannst du KI komplett offline nutzen.

L

LAN (Local Area Network)

Dein lokales Netzwerk zu Hause — alle Geräte, die an deinem Router hängen. Funktioniert auch ohne Internet!

LLM (Large Language Model)

Die "Gehirne" hinter KI-Chatbots wie ChatGPT. Große Sprachmodelle, die auf riesigen Textmengen trainiert wurden. Ollama lässt sie auf deinem Computer laufen.

Technisch
Transformer-basierte neuronale Netzwerke mit Milliarden von Parametern. Beispiele: Llama 3, Mistral, Gemma, Phi. Werden für lokale Nutzung quantisiert (z.B. auf 4-Bit).

LoRa

Eine Funktechnik für sehr weite Strecken (bis 10+ km) bei niedrigem Stromverbrauch. Wird von Meshtastic genutzt für Kommunikation ohne Internet.

M

Mesh-Netzwerk

Ein Netzwerk, bei dem jedes Gerät auch als Weiterleiter fungiert. Wenn ein Gerät ausfällt, finden die Daten einen anderen Weg. Perfekt für Kommunikation ohne Infrastruktur.

N

NAS (Network Attached Storage)

Eine Festplatte, die an dein Netzwerk angeschlossen ist. Alle Geräte im Haushalt können darauf zugreifen — deine eigene private Cloud.

O

Offline-First

Ein Konzept bei dem eine App zuerst offline funktioniert und das Internet nur als Bonus nutzt. Das Gegenteil von "Cloud-First". Genau das Prinzip von Offline Ark!

Open Source

Software, deren Programmcode öffentlich ist. Jeder kann sie prüfen, verbessern und kostenlos nutzen. Alle Tools in Offline Ark sind Open Source.

P

P2P (Peer-to-Peer)

Direkte Verbindung zwischen zwei Geräten, ohne Server dazwischen. LocalSend nutzt P2P um Dateien zu teilen.

Q

Quantisierung

Ein Verfahren, das KI-Modelle kleiner und schneller macht, mit minimalem Qualitätsverlust. Wie MP3 vs. WAV — kleiner, aber klingt fast gleich.

Technisch
Reduktion der Fließkomma-Präzision (z.B. von FP16 auf INT4). Q4_K_M ist ein guter Kompromiss. Q8_0 ist nahezu verlustfrei. GGUF-Format unterstützt verschiedene Quantisierungsstufen.

R

RAM (Arbeitsspeicher)

Der "Kurzzeitgedächtnis" deines Computers. Je mehr RAM, desto größere KI-Modelle kannst du laden. 8 GB ist Minimum, 16 GB ist komfortabel, 32 GB ist ideal.

Repository / Repo

Ein Speicherort für Programmcode, meistens auf GitHub. Das Offline Ark Repository enthält den gesamten Code dieser Webseite.

REST API

Eine verbreitete Art, wie Programme über das Netzwerk miteinander kommunizieren. Ollama bietet eine REST API, damit andere Programme die KI nutzen können.

S

Server

Ein Computer, der Dienste für andere bereitstellt. Dein Raspberry Pi kann ein Server sein, der Wikipedia für alle Geräte in deinem WLAN bereitstellt.

Service Worker

Ein kleines Programm im Browser, das Webseiten offline verfügbar macht. Diese Webseite (Offline Ark) nutzt einen Service Worker!

SSH (Secure Shell)

Eine sichere Verbindung zu einem anderen Computer über das Netzwerk. Wird oft genutzt, um einen Raspberry Pi ohne Bildschirm zu steuern.

Systemd

Das "Management-System" von Linux, das Programme automatisch startet und überwacht. Damit kann Kiwix oder Ollama beim Computerstart automatisch laufen.

T

Terminal

Das Programm, in dem du die Kommandozeile (CLI) nutzt. Windows: PowerShell oder Terminal. macOS: Terminal. Linux: Terminal, Konsole, etc.

Token (KI)

Die "Wortstücke", in die ein KI-Modell Text zerlegt. Ein Token ist ungefähr 3/4 eines Wortes. Mehr Tokens = längere Antworten = mehr Rechenzeit.

U

USB

Der universelle Anschluss für Geräte und Speicher. USB-Sticks sind perfekt, um Offline-Daten wie Wikipedia-ZIM-Dateien oder KeePassXC-Datenbanken mitzunehmen.

V

VPN (Virtual Private Network)

Ein verschlüsselter "Tunnel" durchs Internet. Schützt deine Verbindung, wenn du doch mal online bist. Offline brauchst du kein VPN.

W

WOFF2

Ein komprimiertes Schriftart-Format für Webseiten. Offline Ark nutzt WOFF2-Dateien, damit die Schriften auch ohne Internet geladen werden.

Y

YAML

Ein Textformat für Konfigurationsdateien. Wird oft bei Docker und anderen Tools verwendet. Ähnlich wie JSON, aber einfacher zu lesen.

Z

ZIM (Dateiformat)

Das Dateiformat, das Kiwix nutzt, um komplette Webseiten offline zu speichern. Eine einzige ZIM-Datei kann die ganze Wikipedia enthalten!

Technisch
Zeno IMproved format. LZMA/Zstd-komprimiert. Enthält HTML, Bilder, Metadaten und einen Volltextindex. Unterstützt Cluster-basierte Kompression für effiziente Speicherung.
